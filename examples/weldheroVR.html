Weld Hero VR


Client: Totem Learning
Role:Gameplay Programming

About

Weld Hero was an existing project at Totem, designed to peak interest in welding via a mobile game. I was tasked with adapting the Android based game to work with google cardboard and a playstation controller.
Due to this being an existing project nearing completion a lot of the game's design and assets were already in place with little thought given to 3D. As a result one of the biggest issues I faced was optimisation, not only of the controls (previously touch) but also dealing with the game effectively being rendered twice. To handle the head tracking and multiple cameras I used the Dive plugin.
One of the fundamental things I learnt during this project is the importance of clear vision for the end result, and not adding on features without the proper consideration. A lot of the complications in this project came from the game simply not being designed to work in 3d space, things like the weld gun being a sprite and therefore only represented in 2D space.
The main goal set in this project was to understand programming for Virtual Reality and the implications/impact that has on the games. This project showed me just how young VR is, and in its current iteration, there are crucial design decisions that have to be taken into account. That being said VR is extremely exciting and a new space in the industry which no one has really 'worked out' yet.
If given the opportunity to revisit this project or a similar idea I would create the game from the ground up with VR in mind, that way a lot of the optimisation and design issues would be fixed before the project began.
