About

A mission generation for the game Stars Without Number. Which is an open world stye pen an paper role playing game, and as such in order to breathe more life into the world I wanted to give the players a lot of options at least at first glance for missions. I started scoping a lot of these missions out on paper and then decided I could quite easily automate this with a generator. So I applied my knowledge of Unity, and created a simple generator.
The main hurdle I faced in this project was finding the content, I wanted the generator to have as much variety as possible. This meant that I needed to add as much information as I could find. To overcome this I used XML to hold the information and then added to it over time from various lists and other generators online.
One of the key things I learnt during this project was Inheritance. I had all of the job types extend from a base class and then using overrides I could manipulate their specific details. This allowed me to save a lot of code by having the needed functions in Job, then only having task specifics in the subsequent classes.
I wanted a system at the end of this project that could quickly output a list of Jobs with thier various requirements and needed information. I also wanted to have the framework for extending the system, and as such I made sure to give the classes as much information as possible; For example Planet which is currently just used for location, holds all the information for a planet even if its not currently used.
I would like to extend the generator to represent the ship's computer, so hold the navigation information, dummy email, system status etc. Effectively letting me create a minigame for the players that they can use as their 'comms'. One of the features I plan to work on next is introducing the navigation, which will be based around the hex grid of he universe and from that extend the jobs to pick planets closer to the players current position.
