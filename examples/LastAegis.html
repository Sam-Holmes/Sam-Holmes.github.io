About

The first project that we at Periphery undertook, a tactical roleplaying game set in space. With the recent rise in space games, and the teams passion for space as a setting, we set forth to make Last Aegis. Drawing on inspiration from titles such as Vandal Hearts, Fire Emblem as well as the influence of anime.
The team currently consists of 7 members including contracters. I primarily work on the backend systems (such as XML integration, custom objects etc) to help give the game its foundation. As a result I try to do all my work as modular as possible so that functionallity can be plugged in and out as needed, or improved. In doing so my experience and skillset have evolved tenfold.
When I started this project I wanted to learn how the development cycle works. As such I had to overcome a lot of knowledge barriers. One of the biggest hurdles I faced however was organisation, but I feel this projects has really helped me with both my self motivation and organisation. I used many tools to aid this, such as Trello and the AGILE methodology.
This project is constantly teaching me new ways to approach both the design of my code and the methods I use to implement it. One of the key things I have learnt from this project is the value of Minimal Viable Products, from the simplist system to the most complex, I have found it infinitely useful to prototype out my code and really lock down the core of what I want to acomplish, before adding the layers of complexity.
